# Organize Research Outputs

You are helping organize accumulated research outputs into a logical structure.

## Task

1. Scan the `outputs/` directory for all markdown files
2. Analyze outputs by:
   - Topic/theme
   - Jurisdiction
   - Date
   - Related research questions
3. Propose an organization scheme:
   - Create subdirectories by theme (e.g., outputs/prescription-limits/, outputs/diagnosis-access/)
   - Identify duplicates or redundant content
   - Flag outputs ready for synthesis
   - Identify gaps in research coverage
4. Ask for user approval before reorganizing
5. Execute the reorganization
6. Create an index file (`outputs/INDEX.md`) mapping topics to files

## Organization Principles

- Group related research together
- Maintain chronological ordering within groups
- Preserve original filenames
- Create README files in subdirectories explaining the grouping
- Flag outputs that cite each other for cross-referencing

## Suggested Categories

Based on the research focus, consider organizing by:
- Jurisdiction (UK, US, EU, Israel, etc.)
- Policy aspect (prescription, diagnosis, access, cost)
- Stakeholder (patients, providers, policymakers)
- Temporal (historical analysis, current state, proposals)
